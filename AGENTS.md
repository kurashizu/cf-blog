## Site-Wide Search (Vectorize + Gemini Embedding 2)

| Component | Location | Purpose |
|---|---|---|
| Vectorize index | `blog-search` (768 dims, cosine) | Stores chunk embeddings |
| 3-min cron | `cache-worker/src/handlers/search-index.ts` | Index one dirty item (blog or news) per tick + cleanup stale vectors |
| 30-min cron | `cache-worker/src/lib/refresh.ts` | Marks changed articles dirty via `content_hash` comparison |
| Search API | `cf-blog/app/api/search/route.ts` | `GET /api/search?q=...` â€” embeds query â†’ Vectorize query |
| Search page | `cf-blog/app/search/page.tsx` | SSR search results page |
| Search bar | `cf-blog/components/search/SearchBar.tsx` | Header search icon/input |

### How it works

1. **Blog content change detection** â€” 30-min `refreshCache()` computes `SHA-256(title + excerpt + first 500 chars of body)` for each article. If `content_hash` differs from the stored value, sets `search_updated_at = NULL` (dirty).
2. **News summary â†’ dirty** â€” Heartbeat sets `search_updated_at = NULL` on the news item after generating its summary.
3. **3-min `handleSearchIndexing()`** â€” Each tick:
   - Finds a dirty blog post (blog priority) â†’ reads full markdown from R2 â†’ `chunkItem()` â†’ `batchEmbedContents()` â†’ `upsert()` â†’ marks clean
   - If no dirty blog, finds a dirty news item â†’ same pipeline
   - If nothing dirty â†’ `cleanupSearchIndex()`: reads `cache/search-vectors.json` from R2, compares against current D1 contents, deletes stale vectors
4. **Search API** â€” Embeds query with `"task: search result | query: {q}"` prefix, queries Vectorize with `topK=15`, returns matches with metadata (source, title, heading, excerpt).

### Chunking strategy

Blog and news content both go through `chunkItem()` in `cache-worker/src/lib/chunker.ts`:
- **Overview** chunk (blog only): title + description + tags
- **Section** chunks: split by `##` headings
- **Paragraph** chunks: if a section exceeds 1200 chars, split by `###` â†’ paragraphs â†’ sentences
- Each chunk is formatted: `"title: {title} | text: {content}"` (Gemini Embedding 2 document format)

### Vector metadata schema

```typescript
{
  source: "blog" | "news";
  type: "overview" | "section" | "paragraph";
  id: string;          // blog slug or "news-{id}"
  title: string;
  heading: string;      // section heading (empty string for overview/paragraph)
  excerpt: string;      // ~150 char preview
  tags: string[];       // blog tags
  url: string;          // news original URL
  by: string;           // news author
  published_at: string; // ISO date
}
```

## Cache Worker

`cache-worker/` is a cron-triggered Cloudflare Worker that pre-fetches all external data into R2 + D1. The homepage never calls GitHub / Artificial Analysis on user requests.

| Cache key | Storage | Source | Notes |
|---|---|---|---|
| D1 `github_repos` | D1 | GitHub REST API | All public repos (paginated), fetches per-repo language breakdown (`/repos/*/languages`) for top-3 % display |
| D1 `posts` | D1 | R2 scan of `articles/*.md` â†’ D1 | Parsed frontmatter, sorted by date |
| `cache/github-contributions.json` | R2 | GitHub GraphQL | Needs `GITHUB_PERSONAL_ACCESS_TOKEN` secret |
| `cache/hn-news.json` | R2 + D1 `news_items` | HN API | Latest 5 in R2, full history in D1. Summaries generated by heartbeat, not by 30-min cron |
| `cache/llm-leaderboard.json` | R2 | Artificial Analysis API | Needs `ARTIFICIAL_ANALYSIS_API_KEY` secret |

Schema reference: `database/schema.sql`

Manual trigger:
```bash
curl -X POST https://cf-blog-cache.kurashizu123.workers.dev/__refresh \
  -H "Authorization: Bearer $CRON_SECRET"
curl -X POST https://cf-blog-cache.kurashizu123.workers.dev/__heartbeat \
  -H "Authorization: Bearer $CRON_SECRET"
```
Response is `{ success, logs }` with one `OK/FAILED/SKIPPED` line per cache. Useful for diagnosing cron failures without `wrangler tail`.

### Heartbeat (3-min cron)

A separate `*/3 * * * *` cron processes pending news item rewrites one at a time. Each tick:
1. Queries D1 for a `news_items` row with empty `summary`
2. Fetches the article URL, strips HTML (and extracts image URLs)
3. Calls `gemma-4-31b-it` via Gemini API to produce a full-length Markdown rewrite
4. Response parts with `thought: true` are filtered out; only non-thought content is stored
5. Updates the row's `summary` field

The 30-min cron no longer calls Gemini â€” it only fetches HN items. The `__heartbeat` endpoint also triggers this manually.

## Agent Worker (cf-agent)

Separate Next.js Cloudflare Worker using the same `@opennextjs/cloudflare` build as `cf-blog`. URL: https://agent.022025.xyz. No R2 â€” talks to Gemini directly for chat + tool calling. Default model: `gemma-4-31b-it`.


### Tool System

All tools live under `lib/tools/` and are **manually imported** â€” `fs.readdirSync` does NOT work in Cloudflare Workers.

**To add a new tool:**
1. Create `lib/tools/<tool-name>.ts` implementing the `Tool` interface
2. Import and register in `lib/tools/index.ts` `TOOL_LIST`
3. No other changes â€” `/api/tool` and `/api/chat` auto-discover via the registry

```typescript
interface Tool {
    name: string;            // "eval_expression"
    description: string;     // Human-readable, sent to Gemini
    example: string;         // Example usage JSON
    parameters: object;      // JSON Schema â†’ functionDeclaration
    execute(args: Record<string, unknown>): Promise<unknown>;
}
```

| Tool | Description | Parameters |
|---|---|---|
| `eval_expression` | Safe JS expression evaluator (recursive-descent, no `eval`/`Function`) | `{ code: string }` |
| `web_search` | Brave Search API | `{ q: string }` |
| `get_time` | Current time in timezone (UTC fallback) | `{ timezone?: string }` |
| `blog_search` | Blog semantic search (Vectorize + Gemini Embedding) | `{ q: string, topK?: number, source?: "all" | "blog" | "news" }` |
| `blog_read` | Read full article Markdown by slug | `{ slug: string, maxLength?: number }` |

### API Endpoints

- `GET /api/tool` â€” list all tools with names, descriptions, examples, and Gemini `functionDeclarations`
- `POST /api/tool` â€” execute a tool: `{ name, args }` â†’ `{ success, result, tool }`
- `POST /api/chat` â€” streaming chat with tool-calling loop

**SSE event order** for `/api/chat`:
```
start_process
  â†“ (optional tool iterations, max 5)
tool_start â†’ tool_result â†’ end_tool
  â†“
start_text â†’ text â†’ end_text
  â†“
end_process
```

Think blocks (`<think>...</think>`) are filtered before text is emitted. Session messages stored in KV with 1-hour TTL.

## Hero Data Widgets (cf-blog)

The homepage hero has two data widgets, both transparent (no card chrome):

1. **Contributions ribbon** â€” single horizontal row of 51 cells (14Ă—14 each, day-by-day), shows the user's last ~2 months of GitHub activity. Month labels (Apr/May/Jun) above the cells at month boundaries.
2. **Top languages donut** â€” 100Ă—100 SVG donut in the hero right column, with compact vertical legend. Hover arcs or legend items to see the language name + percentage in the center. Color-coded using GitHub's official language palette.

Contributions powered by `cf-blog-cache` cron â†’ R2, rendered server-side via `getContributions()`. Languages read from D1 `github_repos` table (all non-fork repos, not just top 6) via `getTopLanguages()` from `lib/languages.ts`.

## Conventions

- All Next.js workers use `@opennextjs/cloudflare` (not `next start`).
- Static assets go in `public/`. R2-bound content (article markdown) lives in R2, not the repo.
- Theme colors via CSS variables in `components/theme/global.css`. Tailwind utilities map to these (`bg-bg-card`, `text-text-muted`, etc.).
- Component files: one per file, filename matches component name. `"use client"` only when needed.
- For R2 operations, use `wrangler r2 object` (the Cloudflare MCP API has known issues).
- Delete obsolete code completely â€” no commented-out blocks, no `_unused` files, no half-removed components. If a file is dead, `git rm` it.

### Animation System

- **Page transition**: `PageTransition` in layout (`components/layout/PageTransition.tsx`) handles route-change fade-in. On navigation, content hides instantly (`opacity: 0`, no transition) then fades in via double `requestAnimationFrame`. No exit animation â€” App Router doesn't support it reliably.
- **Element entrance**: All staggered entrance classes (`animate-fade-up`, `animate-fade-up-sm`, `animate-fade-zoom`, `animate-fadeIn`, `animate-scaleIn`, `animate-slideUp`, `animate-slideDown`) use `ease-out` at 0.18â€“0.3s. No spring/scale.
- **VisitorTerminal cursor**: Static accent-colored block, no blink. Typewriter effect at 16ms/char.
